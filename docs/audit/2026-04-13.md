# Security Audit — 2026-04-13

Comprehensive 0-day audit of the Cyrius port (3,643 lines, 20 modules).

## Critical Findings (Fix Immediately)

### C1: Command bypass via absolute/relative paths
**Severity: CRITICAL** | `permissions.cyr:is_blocked_command()`

Blocked/admin command lists only match basenames (`streq(cmd, "dd")`).
Trivially bypassed with `/usr/bin/dd`, `./rm`, `../bin/chmod`.

**Attack:** `execute /usr/bin/dd if=/dev/sda of=/tmp/disk.img` → bypasses all checks.

**Fix:** Extract basename before classification. Resolve symlinks.

### C2: Unsanitized args passed to execve
**Severity: CRITICAL** | `translate.cyr:translate_*()`, `security.cyr:execute_command()`

User-controlled fields from NL parsing flow directly to command args with no
metacharacter validation. Shell injection possible if subprocess spawner
interprets `;`, `&&`, `$()`, backticks.

**Fix:** Validate all args against dangerous char set: `; | & $ ( ) < > \` ' " \\ \n`

### C3: Null pointer dereference in translators
**Severity: CRITICAL** | `translate.cyr:translate_show_file()`, `translate_change_dir()`, `translate_create_dir()`, `translate_remove()`

Field extraction (`extract_last_arg`, `extract_after`) returns 0 on failure.
Multiple translators push field1/field2 to args without null checks.
`translate_remove()` pushes null to `rm` args.

**Fix:** Null-guard all field access in every translate function.

### C4: Audit log JSON injection
**Severity: CRITICAL** | `audit.cyr:AuditEntry_to_json()`

User input flows into `json_set_str()` without escaping quotes/backslashes.
Attacker can forge audit entries, spoof identity, corrupt forensic trail.

**Attack:** Input `show "}, "user": "admin", "approved": 1, {"x":"` rewrites the audit record.

**Fix:** Escape `"`, `\`, newlines in all JSON string values.

### C5: Unhandled intent tags in translator
**Severity: HIGH** | `interpreter.cyr:Interpreter_translate()`

Parser creates AUDIT_VIEW(16), AGENT_INFO(17), SERVICE_CONTROL(18),
DISK_USAGE(13), PIPELINE(208) intents but translator has no match cases.
All fall to default `translate_unknown()` → SAFE permission echo.

**Attack:** "show audit log" → classified as SAFE instead of requiring admin approval.

**Fix:** Add match cases for all parsed intent tags, or return error for unhandled.

## High Findings

### H1: SUID/EUID bypass in root detection
**Severity: HIGH** | `security.cyr:SecurityContext_new()`

Checks `uid == 0` but not `euid == 0`. Setuid agnoshi binary would run with
root effective privileges but without restricted mode enforcement.

**Fix:** Check `euid == 0` in addition to `uid == 0`.

### H2: Environment inheritance in privilege escalation
**Severity: HIGH** | `security.cyr:execute_with_privileges()`

Child process inherits parent environment. `LD_PRELOAD`, `LD_LIBRARY_PATH`,
`PATH` can be set to hijack sudo or the escalated command.

**Fix:** Exec with clean environment whitelist: `PATH=/usr/bin:/bin:/usr/sbin:/sbin`

### H3: Checkpoint dir in world-readable /tmp
**Severity: HIGH** | `checkpoint.cyr:CheckpointManager_new()`

Checkpoints stored in `/tmp/agnoshi-checkpoints` — world-readable, symlink-attackable.
Backed-up file contents (from rm undo) are exposed to other users.
TOCTOU race between mkdir and file creation.

**Fix:** Use `$HOME/.agnoshi/checkpoints` with 0700 perms. Use `O_NOFOLLOW|O_EXCL`.

### H4: Git branch name terminal injection
**Severity: HIGH** | `prompt.cyr:prompt_detect_git_branch()`

`.git/HEAD` content rendered directly in prompt without sanitization.
Malicious branch names with ANSI escape sequences can manipulate terminal display.

**Fix:** Strip control characters (< 0x20 and 0x7F) from branch name.

### H5: Approval UI terminal escape injection
**Severity: HIGH** | `approval.cyr:ApprovalManager_request()`

Command name and args displayed via `print_str()` without escaping.
Attacker crafts command with embedded ANSI sequences to fake the approval prompt,
hide the real command, or auto-type 'a' (approve).

**Fix:** Use `print_str_safe()` that strips control characters for all approval display.

### H6: History file symlink attack
**Severity: HIGH** | `history.cyr:CommandHistory_save()`

TOCTOU between directory creation and file write. Permissions set AFTER write.
Attacker can symlink history path to `/etc/passwd` before write completes.

**Fix:** Use `O_NOFOLLOW|O_EXCL` for file creation. Set perms before write via `fchmod`.

### H7: Git commit message argument injection
**Severity: HIGH** | `translate.cyr:translate_git_commit()`

Commit message extracted from NL input added as `-m` argument without validation.
Leading `-` in message is interpreted as git flags. Shell metacharacters possible.

**Fix:** Validate message doesn't start with `-`. Sanitize shell metacharacters.

## Medium Findings

### M1: /proc/self/environ 8KB buffer
**Severity: MEDIUM** | `main.cyr:getenv()`

Fixed 8KB buffer for /proc/self/environ. Truncates silently if env > 8KB.
Potential off-by-one in null-delimiter parsing.

**Fix:** Use dynamic allocation or increase buffer. Validate bounds.

### M2: kill PID=0 via str_to_int
**Severity: MEDIUM** | `interpreter.cyr` (KILL_PROCESS parse)

`str_to_int("foobar")` returns 0. `kill 0` sends signal to entire process group.

**Fix:** Validate PID > 0 after conversion.

### M3: rm flag check incomplete
**Severity: MEDIUM** | `permissions.cyr:rm_has_dangerous_flags()`

Doesn't handle `--` (stops flag processing), combined flags like `-fir`,
or flags after file args (`rm file -r`).

**Fix:** Parse all `-` prefixed args for dangerous chars. Block `--`.

### M4: Path traversal in file operations
**Severity: MEDIUM** | `translate.cyr:translate_show_file()`, `translate_find_files()`

No `../` validation on paths. User can read arbitrary files via
`read ../../../etc/shadow`.

**Fix:** Block paths containing `..`. Consider realpath validation.

### M5: Command parsing doesn't handle escape sequences
**Severity: MEDIUM** | `commands.cyr:split_command_line()`

Quote parser doesn't handle `\"` escape. Can break out of quoted args.

**Fix:** Handle backslash escapes in quoted strings.

### M6: Alias expansion injection
**Severity: MEDIUM** | `aliases.cyr:AliasManager_expand()`

Alias expansion is string concatenation. Malicious alias values can inject
shell metacharacters. Approval shows unexpanded alias.

**Fix:** Validate alias values against metacharacter set.

### M7: Checkpoint doesn't block execution on failure
**Severity: MEDIUM** | `session.cyr:Session_execute_shell()`

If `CheckpointManager_checkpoint_remove()` fails (returns 0), execution
proceeds anyway. Undo won't work for this operation.

**Fix:** Warn user or block execution if checkpoint fails for destructive ops.

### M8: /etc/passwd username not validated
**Severity: MEDIUM** | `security.cyr:security_get_username()`

Returned username not validated for safe characters. Special chars in
username field could be exploited if used in shell contexts later.

**Fix:** Validate username matches `[a-zA-Z0-9_-]`.

### M9: TOCTOU in sudo availability check
**Severity: MEDIUM** | `security.cyr:security_check_sudo()`

Checks file existence at init, not at escalation time.
Sudo binary could be replaced between check and use.

**Fix:** Re-verify sudo path, ownership, and permissions at escalation time.
