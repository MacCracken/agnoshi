> **Scope**: Pre-release hardening cycle per the agnoshi `CLAUDE.md` P(-1) work loop and the AGNOS [first-party standards](https://github.com/MacCracken/agnosticos/blob/main/docs/development/planning/first-party-standards.md) "Security Hardening (required before every release)" section. >
> **Author**: agnoshi maintainer pass, automated where possible.
>
> **Status**: In progress. Findings consolidated as each slice lands.

## Methodology

Walk every roadmap-listed P(-1) bullet, produce a finding per bullet (clean / fixed / deferred). Each finding documents:
- **Scope** — what was reviewed
- **Tooling** — how the review was conducted (grep, linter, manual)
- **Findings** — bugs surfaced, severity, fix
- **Forward shield** — what was added to CI/lint to prevent recurrence

Severity: **CRITICAL** (remote / privilege escalation) / **HIGH** (moderate effort, data leak) / **MEDIUM** (specific conditions) / **LOW** (defense-in-depth). Matches the AGNOS standard.

## §2 — Cyrius cstring/Str type-mismatch class (slice 2, committed)

**Scope**: Five distinct type-mismatch bug patterns surfaced over v1.2.0/v1.3.0:
- `str_len(cstring)` (slice 1 v1.2.0)
- `str_sub(start, end)` semantics drift (slice 1 v1.2.0)
- `str_cat(cstring, *)` (v1.2.0 slice 7 + v1.3.0 slice 8)
- `str_cat(*, cstring)` (v1.3.1 slice 2)
- `is_safe_path(Str)` mismatch (v1.3.0 slice 3 — silent translate_unknown for every NL filesystem op since v1.0)

**Tooling**: New static analyzer `scripts/lint-cstr-str.sh` wired into CI.

**Findings on first run**:
- `session.cyr:101` — `str_starts_with(trimmed, "mode ")`: Str-typed fn called with cstring. **HIGH** (silent runtime fallthrough); fixed by wrapping with `str_from`.
- `checkpoint.cyr:22` — `syscall(SYS_CHMOD, ...)`: cross-arch break. **HIGH** (aarch64 build failure); fixed by switching to `sys_chmod` wrapper.
- `main.cyr:32` — legacy pre-port entry, never linked; whitelisted with `# lint:cstr-ok`.

**Forward shield**: 11 lint patterns total (Category A: 5 first-arg-cstring patterns; Category B: 3 second-arg-cstring patterns; Category C: 3 aarch64-broken syscall patterns). Word-anchored regexes (so `cstr_starts_with` doesn't false-match `str_starts_with`). Escape hatch: `# lint:cstr-ok` trailing comment.

## §3 — Buffer safety (slice 3, committed)

**Scope**: All `var buf[N]` static-data declarations in `src/`. Two concerns:
1. Sizing — does the buffer fit its largest write?
2. Escape — does any function return a pointer (Str header) borrowing the local buf?

**Tooling**: Existing CI security-scan flags `var buf[N]` where `N >= 8192` (warn) or `N >= 65536` (fail). Manual catalog of every site. Linter extended with Category D (escape via return / store).

**Findings**:

Verified Cyrius `var buf[N]` allocates **N i64-words (8N bytes)**, not N raw bytes. Confirmed by cross-referencing `lib/process.cyr`'s `var status_buf[1]` against the kernel's 4-byte wstatus write (works → `[1]` ≥ 4 bytes → i64-word).

| Site | Decl | Bytes | Usage | Verdict |
|---|---|---|---|---|
| `agnsh.cyr:230` | `var b[1]` | 8 | read 1 byte/iter | ✓ |
| `agnsh.cyr:273` | `var buf[4096]` | 32K | read up to 4096B | ✓ (28K headroom) |
| `interpreter.cyr:528` | `var cmd_out[1]` | 8 | cstring ptr | ✓ |
| `approval.cyr:85` | `var buf[64]` | 512 | read up to 63B | ✓ |
| `security.cyr:86` | `var stat_buf[18]` | 144 | x86 `struct stat` | ✓ (exact fit) |
| `ui.cyr:3` | `var buf[4096]` | 32K | deferred, read 4095B | ✓ |
| `prompt.cyr:14` | `var hostname_buf[256]` | 2K | uname struct | ✓ |
| `session.cyr:20,138,157,251` | `var cwd_buf[4096]` | 32K | getcwd | ✓ |

All sizes safe.

**Escape findings** (Category D patterns — Str borrowing static buf, lifetime escape):
- `ui.cyr:13` — `return str_from(&buf)` from `read_input_line`. **HIGH** (every caller sees the same Str data after next call); fixed with `str_clone`.
- `prompt.cyr:18` — `str_from(&hostname_buf + 65)` stored in long-lived PromptContext. **HIGH**; fixed with `str_clone`.
- `session.cyr:22, 162, 256` — three `str_from(&cwd_buf)` sites storing cwd into prompt/session structs. **HIGH** × 3; fixed with `str_clone`.

All five are in deferred modules (not in agnsh's include graph yet). Same shape that bit `CommandHistory_add` in slice 7 v1.3.0 (where it WAS live and the symptom was visible-garbled history entries).

**Forward shield**: Linter Categories D added — flags `return str_from(&...)` and `store64(*, str_from(&...))` patterns.

## §4 — Syscall return-value handling (slice 4, this slice)

**Scope**: Every direct `syscall(SYS_*, ...)` and `sys_X` wrapper call in live src/.

**Tooling**: Manual walk of 35+ syscall sites; categorize by return-value handling.

**Categories**:

| Category | Sites | Verdict |
|---|---|---|
| stdout/stderr writes (sanitize, approval, security, ui) | 8 | LOW — terminal/pipe failures rare; conventional to ignore; not fixed |
| Process exit (agnsh:404) | 1 | ✓ — exit syscall never returns |
| getuid/getgid/geteuid (security) | 3 | ✓ — never fail in Linux |
| stdin reads (read_line, approval, ui) | 3 | ✓ — return checked |
| chdir (session) | 2 | ✓ — return checked |
| uname (prompt) | 1 | ✓ — return checked |
| sys_open (audit) | 1 | ✓ — fd checked |
| sys_stat (security) | 1 | ✓ — return checked |
| **sys_chmod (security-critical) — UNCHECKED** | **2** | **HIGH** — fixed |
| getcwd (session) — UNCHECKED | 3 | MEDIUM — deferred to v1.4.0 wire-up |
| sys_write/sys_close in audit logger | 3 | LOW — best-effort logging; fd already validated |

**Findings**:

- **`history.cyr:112`** — `sys_chmod(load64(self + 24), 384)` on `$HOME/.agnsh_history`. Return ignored. **HIGH severity** — if chmod fails (race / NFS / unusual filesystem), the history file stays at umask default (typically 0644), leaking every shell command the user has ever typed to other users on a multi-user system. **Fixed**: capture return, write stderr warning on failure. The file write itself already succeeded so save() still returns 0; the operator gets a visible signal.
- **`checkpoint.cyr:29`** — `sys_chmod(str_data(dir), 448)` on `$HOME/.agnoshi/checkpoints/`. Same shape, holds pre-destructive-op backups. **HIGH severity**; same fix applied.
- **`session.cyr:21, 161, 255`** — `syscall(SYS_GETCWD, &cwd_buf, 4095)` × 3 sites, return ignored. **MEDIUM severity** — on failure cwd_buf contains stale or zero bytes, then `str_clone(str_from(&cwd_buf))` wraps garbage as a "cwd" Str that gets stored in the prompt context. Deferred to v1.4.0 wire-up; module not yet in agnsh include graph and the fix involves a fallback-path decision (display "?" or last-known cwd or fail-loudly).

**Forward shield**: Linter Category E — flags any `sys_chmod(...)` statement where the result isn't captured into a variable or an `if` condition. Pattern: `^\s*sys_chmod\(` (start-of-line whitespace + bare call). Matches `    sys_chmod(path, 384);` but not `var rc = sys_chmod(...)` or `if (sys_chmod(...) != 0)`. CI gate.

## §6 — Input validation sweep (slice 6, committed)

**Scope**: All external-input entry points — argv, stdin, env, file reads.

**Tooling**: Manual catalog + grep.

**Categories**:

| Source | Sites | Verdict |
|---|---|---|
| `argv()` | `agnsh.cyr:379,395` | ✓ Routes through streq/parse. CLI args are user-controlled; bounded by Linux ARG_MAX (~2 MB). |
| stdin reads | `approval.cyr:86` (63B max), `agnsh.cyr:232` (1B/iter, 4095B total), `ui.cyr:4` (4095B) | ✓ All bounded. |
| `getenv()` | `agnsh.cyr` (audit_log_path, history_path), `prompt.cyr`, `sanitize.cyr` (build_safe_env), `checkpoint.cyr`, `session.cyr` | ⚠ Trust-env-source; values unvalidated but used in $HOME-rooted path construction. Practical impact zero (user owns env); flagged as a defense-in-depth gap. |
| `file_read_all` | `security.cyr` (`/etc/passwd`), `history.cyr` (history file), `prompt.cyr` (.git/HEAD) | ✓ All bounded with explicit buf+maxlen. |

**Findings**:
- **prompt.cyr — three stale-stdlib build breaks**. Caught while sweeping file_read_all sites: `fs_exists`, single-arg `file_read_all`, `fs_parent` — all renamed/removed in Cyrius 5.10.x. Would have build-broken the v1.4.0 wire-up. **Fixed**: rewrote `prompt_detect_git_branch` to use `file_exists` + buffer-based `file_read_all` + manual cstring path build (mirrors `agnsh.cyr::audit_log_path` shape). Parent-walk via `fs_parent` replaced with a single-iteration break (TODO for v1.4.0: implement dirname-style truncate on the Str's data; needs cstring-friendly path representation through the prompt module).
- **prompt.cyr — `str_starts_with(content, prefix)` where `prefix = "ref: refs/heads/"`** — variable holding a cstring literal passed to a Str-only fn. Linter's literal-arg grep doesn't catch the via-variable case. **Fixed**: wrapped declaration with `str_from`. Documented as a known linter gap (would need multi-line analysis to grep for `var X = "literal"` + downstream `str_X(*, X)` use).

**Forward shield**: prompt.cyr's `prompt_detect_git_branch` now ready for v1.4.0 wire-up. The variable-cstring linter-gap is documented; if a third instance surfaces, that earns its own bite.

## §7 — Path-traversal sweep (slice 7, committed)

**Scope**: Every file-op site (sys_open, file_exists, file_read_all, file_write_all, sys_stat, sys_chmod, SYS_CHDIR).

**Tooling**: Manual grep over `src/*.cyr`.

| Site | Path source | Verdict |
|---|---|---|
| `audit.cyr:54 sys_open` | `audit_log_path()` → `$HOME/.agnsh_audit.log` | ✓ env-rooted; user owns env |
| `history.cyr:14,21,109,118` | `history_path()` → `$HOME/.agnsh_history` | ✓ same shape |
| `prompt.cyr:71,73` | `<cwd>/.git/HEAD` | ✓ cwd from kernel getcwd |
| `security.cyr:42,85,87,99,100,153,154` | `/etc/passwd`, `/usr/bin/sudo`, `/bin/sudo` literals | ✓ hardcoded |
| `checkpoint.cyr:32` | `$HOME/.agnoshi/checkpoints` from str_cat | ⚠ Deferred — passes `str_data(Str)` (non-null-terminated) to syscall; same shape as session findings below |
| `session.cyr:158,252` | `chdir(dir)` where dir is parser-extracted Str | ✓ chdir traversal is intentional (cd ../foo is a feature) BUT same str_data(Str) null-termination gap |
| `translate.cyr` (11 sites) | parser-extracted Str fields | ✓ All routed through `safe_path_in_str` / `safe_arg_in_str` |

**Findings**:
- **No live-binary path-traversal exposures.** Every file-op site is either: literal-path (hardcoded), env-rooted (user-owns), parser-extracted-and-validated (safe_path_in_str), or kernel-trusted (getcwd).
- **`str_data(Str)` → syscall path is a separate bug class** — `session.cyr:158, 252 SYS_CHDIR` and `checkpoint.cyr:32 sys_chmod` both pass `str_data(Str)` to a syscall expecting a null-terminated cstring. `str_cat`'s output is NOT null-terminated; the kernel reads past the buffer until it finds a zero. **MEDIUM severity** (could read attacker-controlled bytes from adjacent memory and use them as part of the path). Deferred — both modules are not in agnsh's include graph; the v1.4.0 wire-up slice needs to either (a) build cstring paths the same way `agnsh.cyr::audit_log_path` does, or (b) add a `str_to_cstring(s)` helper that allocates a null-terminated copy.

**Forward shield**: Could extend the linter to flag `syscall(SYS_X, str_data(...))` and `sys_X(str_data(...), ...)` patterns. **NOT added this cycle** — the false-positive rate is hard to predict (some syscalls take ptr+len explicitly, e.g., `sys_write`). Re-evaluate at v1.4.0 wire-up.

## §8 — Known-CVE / pattern review (slice 8, committed)

**Scope**: Walk well-known CVE patterns against agnoshi's surface.

**Tooling**: Manual review against the CVE-pattern checklist in AGNOS first-party standards §Security Hardening.

| Pattern | agnoshi surface | Verdict |
|---|---|---|
| Format-string bugs | No printf-family in agnoshi (uses str_print + str_builder) | ✓ |
| Stack/heap buffer overflow | All `var buf[N]` sized appropriately (see §3) | ✓ |
| Integer overflow in allocation size | Allocations are `alloc(strlen + small_constant)`; strlen ≤ 2^31; no overflow on 64-bit | ✓ |
| Use-after-free | Cyrius arena-allocated; no manual free in agnoshi | ✓ |
| Command injection | No exec calls in live binary; future exec wire-up uses `exec_vec(argv)` (explicit argv, not concatenated cmdline) | ✓ |
| TOCTOU on sudo | `verify_sudo_path` re-checks at escalation moment, but fork+exec window still exists | ⚠ Acknowledged in ADR-006; trade-off documented |
| Symlink races on O_CREAT | audit/history open without `O_NOFOLLOW` | ⚠ See finding below |
| Audit-log injection (JSON) | `json_escape` covers (audit C4 fix from v1.0); locked in slice 4 v1.3.0 tests | ✓ |
| Argument injection (`-` prefix) | `is_safe_commit_message` rejects leading-`-`; locked in slice 6 v1.2.0 tests | ✓ |

**Findings**:
- **Audit-log + history open without `O_NOFOLLOW`** — `agnsh.cyr::audit_one_shot` opens with flags `1089` = `O_WRONLY | O_CREAT | O_APPEND` (no `O_NOFOLLOW`); `history.cyr::CommandHistory_save` uses `file_write_all` which expands to `O_WRONLY | O_CREAT | O_TRUNC`. If an attacker places a symlink at `$HOME/.agnsh_audit.log` or `$HOME/.agnsh_history` before agnoshi runs, the open follows the symlink and overwrites the target. **Severity LOW**: requires pre-existing attacker write access to `$HOME` (unusual on single-user systems; possible on shared-NFS / container setup phases). **Mitigation deferred**: `O_NOFOLLOW` flag value differs per arch (`0o400000` on x86_64, `0o100000` on aarch64-generic); adding it cleanly needs a per-arch constant + `lib/io.cyr` API extension to thread custom flags through `file_write_all`. Queued for v1.4.0.

**Forward shield**: None this cycle. The mitigation cost is non-trivial vs the LOW severity; document and defer.
