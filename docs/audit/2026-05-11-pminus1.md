## Methodology

Walk every roadmap-listed P(-1) bullet, produce a finding per bullet (clean / fixed / deferred). Each finding documents:
- **Scope** — what was reviewed
- **Tooling** — how the review was conducted (grep, linter, manual)
- **Findings** — bugs surfaced, severity, fix
- **Forward shield** — what was added to CI/lint to prevent recurrence

Severity: **CRITICAL** (remote / privilege escalation) / **HIGH** (moderate effort, data leak) / **MEDIUM** (specific conditions) / **LOW** (defense-in-depth). Matches the AGNOS standard.

## §2 — Cyrius cstring/Str type-mismatch class (slice 2, committed)

**Scope**: Five distinct type-mismatch bug patterns surfaced over v1.2.0/v1.3.0:
- `str_len(cstring)` (slice 1 v1.2.0)
- `str_sub(start, end)` semantics drift (slice 1 v1.2.0)
- `str_cat(cstring, *)` (v1.2.0 slice 7 + v1.3.0 slice 8)
- `str_cat(*, cstring)` (v1.3.1 slice 2)
- `is_safe_path(Str)` mismatch (v1.3.0 slice 3 — silent translate_unknown for every NL filesystem op since v1.0)

**Tooling**: New static analyzer `scripts/lint-cstr-str.sh` wired into CI.

**Findings on first run**:
- `session.cyr:101` — `str_starts_with(trimmed, "mode ")`: Str-typed fn called with cstring. **HIGH** (silent runtime fallthrough); fixed by wrapping with `str_from`.
- `checkpoint.cyr:22` — `syscall(SYS_CHMOD, ...)`: cross-arch break. **HIGH** (aarch64 build failure); fixed by switching to `sys_chmod` wrapper.
- `main.cyr:32` — legacy pre-port entry, never linked; whitelisted with `# lint:cstr-ok`.

**Forward shield**: 11 lint patterns total (Category A: 5 first-arg-cstring patterns; Category B: 3 second-arg-cstring patterns; Category C: 3 aarch64-broken syscall patterns). Word-anchored regexes (so `cstr_starts_with` doesn't false-match `str_starts_with`). Escape hatch: `# lint:cstr-ok` trailing comment.

## §3 — Buffer safety (slice 3, committed)

**Scope**: All `var buf[N]` static-data declarations in `src/`. Two concerns:
1. Sizing — does the buffer fit its largest write?
2. Escape — does any function return a pointer (Str header) borrowing the local buf?

**Tooling**: Existing CI security-scan flags `var buf[N]` where `N >= 8192` (warn) or `N >= 65536` (fail). Manual catalog of every site. Linter extended with Category D (escape via return / store).

**Findings**:

Verified Cyrius `var buf[N]` allocates **N i64-words (8N bytes)**, not N raw bytes. Confirmed by cross-referencing `lib/process.cyr`'s `var status_buf[1]` against the kernel's 4-byte wstatus write (works → `[1]` ≥ 4 bytes → i64-word).

| Site | Decl | Bytes | Usage | Verdict |
|---|---|---|---|---|
| `agnsh.cyr:230` | `var b[1]` | 8 | read 1 byte/iter | ✓ |
| `agnsh.cyr:273` | `var buf[4096]` | 32K | read up to 4096B | ✓ (28K headroom) |
| `interpreter.cyr:528` | `var cmd_out[1]` | 8 | cstring ptr | ✓ |
| `approval.cyr:85` | `var buf[64]` | 512 | read up to 63B | ✓ |
| `security.cyr:86` | `var stat_buf[18]` | 144 | x86 `struct stat` | ✓ (exact fit) |
| `ui.cyr:3` | `var buf[4096]` | 32K | deferred, read 4095B | ✓ |
| `prompt.cyr:14` | `var hostname_buf[256]` | 2K | uname struct | ✓ |
| `session.cyr:20,138,157,251` | `var cwd_buf[4096]` | 32K | getcwd | ✓ |

All sizes safe.

**Escape findings** (Category D patterns — Str borrowing static buf, lifetime escape):
- `ui.cyr:13` — `return str_from(&buf)` from `read_input_line`. **HIGH** (every caller sees the same Str data after next call); fixed with `str_clone`.
- `prompt.cyr:18` — `str_from(&hostname_buf + 65)` stored in long-lived PromptContext. **HIGH**; fixed with `str_clone`.
- `session.cyr:22, 162, 256` — three `str_from(&cwd_buf)` sites storing cwd into prompt/session structs. **HIGH** × 3; fixed with `str_clone`.

All five are in deferred modules (not in agnsh's include graph yet). Same shape that bit `CommandHistory_add` in slice 7 v1.3.0 (where it WAS live and the symptom was visible-garbled history entries).

**Forward shield**: Linter Categories D added — flags `return str_from(&...)` and `store64(*, str_from(&...))` patterns.

## §4 — Syscall return-value handling (slice 4, this slice)

**Scope**: Every direct `syscall(SYS_*, ...)` and `sys_X` wrapper call in live src/.

**Tooling**: Manual walk of 35+ syscall sites; categorize by return-value handling.

**Categories**:

| Category | Sites | Verdict |
|---|---|---|
| stdout/stderr writes (sanitize, approval, security, ui) | 8 | LOW — terminal/pipe failures rare; conventional to ignore; not fixed |
| Process exit (agnsh:404) | 1 | ✓ — exit syscall never returns |
| getuid/getgid/geteuid (security) | 3 | ✓ — never fail in Linux |
| stdin reads (read_line, approval, ui) | 3 | ✓ — return checked |
| chdir (session) | 2 | ✓ — return checked |
| uname (prompt) | 1 | ✓ — return checked |
| sys_open (audit) | 1 | ✓ — fd checked |
| sys_stat (security) | 1 | ✓ — return checked |
| **sys_chmod (security-critical) — UNCHECKED** | **2** | **HIGH** — fixed |
| getcwd (session) — UNCHECKED | 3 | MEDIUM — deferred to v1.4.0 wire-up |
| sys_write/sys_close in audit logger | 3 | LOW — best-effort logging; fd already validated |

**Findings**:

- **`history.cyr:112`** — `sys_chmod(load64(self + 24), 384)` on `$HOME/.agnsh_history`. Return ignored. **HIGH severity** — if chmod fails (race / NFS / unusual filesystem), the history file stays at umask default (typically 0644), leaking every shell command the user has ever typed to other users on a multi-user system. **Fixed**: capture return, write stderr warning on failure. The file write itself already succeeded so save() still returns 0; the operator gets a visible signal.
- **`checkpoint.cyr:29`** — `sys_chmod(str_data(dir), 448)` on `$HOME/.agnoshi/checkpoints/`. Same shape, holds pre-destructive-op backups. **HIGH severity**; same fix applied.
- **`session.cyr:21, 161, 255`** — `syscall(SYS_GETCWD, &cwd_buf, 4095)` × 3 sites, return ignored. **MEDIUM severity** — on failure cwd_buf contains stale or zero bytes, then `str_clone(str_from(&cwd_buf))` wraps garbage as a "cwd" Str that gets stored in the prompt context. Deferred to v1.4.0 wire-up; module not yet in agnsh include graph and the fix involves a fallback-path decision (display "?" or last-known cwd or fail-loudly).

**Forward shield**: Linter Category E — flags any `sys_chmod(...)` statement where the result isn't captured into a variable or an `if` condition. Pattern: `^\s*sys_chmod\(` (start-of-line whitespace + bare call). Matches `    sys_chmod(path, 384);` but not `var rc = sys_chmod(...)` or `if (sys_chmod(...) != 0)`. CI gate.
